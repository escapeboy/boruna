## What worked

- **Discovery before implementation cut Sprint A scope ~80%.** Reading `crates/llmvm/src/capability_gateway.rs` first revealed the structured `Policy` already existed — the gap was purely the MCP surface, not the VM. Saved an entire sprint.
- **Skipping formal Think/Plan ceremony for trivial fixes.** The polish pass (tool description + runner docs) was 11 lines; running full sprint orchestration on it would have been process theater. Right call to do it inline.
- **Branch-per-sprint + `--no-ff` merge.** Kept the two sprints disjoint and reviewable in `git log --first-parent`. Single conflict was on CHANGELOG `[Unreleased]` (expected and trivial).
- **Hand-written JSON Schema instead of pulling `schemars` into `boruna-vm`.** Avoided a load-bearing dependency in the core crate; FleetQ gets exactly the schema artifact they asked for.

## What didn't work

- **Test plan drafted with `.ax`-source-level capability tests, then had to be revised** when I discovered `.ax` source can't directly emit `CapCall` — capability calls happen via Effect system. Honest revision saved face but cost a doc rewrite. Lesson: verify the language surface *before* writing test scenarios, not after.
- **Forgot the MCP `#[tool(description=...)]` string** on the first pass. Updated `RunParams` doc-string but the tool description AI agents actually read was untouched. Caught only at the "anything unfinished?" audit. Lesson: when changing a public surface, grep for *every* place it's described, not just the structural one.
- **Pushed to master without considering remote state.** First push rejected because I didn't `git fetch` first. Cost a forced merge commit (`76a9c1d Merge branch 'master'`). Cheap mistake, but a `git fetch` at the start of the sync skill would have prevented it.

## What I'd do differently next sprint

1. **Always `git fetch origin` before starting branch work** — gives me an accurate "ahead/behind" picture from the start.
2. **When changing an MCP tool, audit `#[tool(description=...)]` before commit.** Add to the personal mental checklist alongside CHANGELOG / README updates.
3. **For trivial single-file polish, skip sprint ceremony explicitly** rather than apologizing for it. The skill is for non-trivial changes; trivial = direct commit on master with a clear message.
