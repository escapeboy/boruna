# Boruna VM (`boruna-vm`, dir `crates/llmvm/`) — Research Report

Read-only research. Every claim cites `path:line` against the actual source read this session.
Scope: the virtual machine crate — execution loop, capability gateway/Policy, actor system, replay/EventLog,
net record/replay, HTTP handler (`http` feature), errors, value ops.

---

## 1. Purpose & Architecture

`boruna-vm` executes compiled bytecode (`boruna_bytecode::Module`) on a stack machine and is the
single choke point where side effects are gated. Design:

- **Stack VM** (`vm.rs`): a `Vm` holds an operand `stack`, a `call_stack` of `CallFrame`s (each with
  its own `locals`), `globals`, a `CapabilityGateway`, and an `EventLog`. The main loop `execute()`
  (`vm.rs:326`) dispatches one `Op` per iteration, incrementing `step_count` each step.
- **Two execution entry modes**: `run()` (`vm.rs:205`, unbounded-until-`max_steps`) and
  `execute_bounded(budget)` (`vm.rs:227`, preemptive slicing → `StepResult::{Completed,Yielded,Blocked,Error}`),
  the latter driven by the actor scheduler and the MCP streaming path.
- **Capability gating** (`capability_gateway.rs`): every side-effecting operation is an `Op::CapCall`
  routed through `CapabilityGateway::call()` (`capability_gateway.rs:468`), which enforces a `Policy`
  (allow/deny + per-cap budget) and dispatches to a pluggable `CapabilityHandler`
  (`MockHandler`, `HttpHandler`, `StepInputHandler`, `LlmRouterHandler`, `Replay*`).
- **Actor system** (`actor.rs`): deterministic round-robin scheduler over `Vm` instances, with
  bounded rounds/budget, OneForOne crash cascade, and deterministic message ordering.
- **Replay** (`replay.rs`): `EventLog` records `CapCall`/`CapResult`/actor/scheduler/UI events;
  `ReplayEngine::verify` / `verify_full` compare original vs replayed logs.
- **Network** (`http_handler.rs`, `net_record_replay.rs`): `http`-feature-gated real HTTP with SSRF
  guards and NetPolicy allowlists, plus a record/replay tape layer for determinism.

Threat model note: `Module::from_json` / `Module::from_bytes` (`crates/llmbc/src/module.rs:100,121`)
deserialize modules with **no structural/index validation**, so "crafted bytecode" is a real input
surface, not hypothetical. The VM defends most of it with typed errors (see §3/§4).

## 3. GAPS

- **`actor.rs:225` — `.expect()` panic on crafted `SpawnActor` (HIGH).**
  `child_vm.set_entry_function(req.func_idx).expect("invalid function index in spawn request")`.
  `func_idx` originates from `Op::SpawnActor(func_idx)` in bytecode (`vm.rs:526`). Since modules load
  without index validation, a module whose `SpawnActor` targets an out-of-range function makes
  `set_entry_function`→`call_function` return `Err(VmError::InvalidFunction)` (`vm.rs:302`), which
  `.expect()` turns into a process panic → DoS. This is the **only** production-code `.expect()` in
  the crate (grep: all other unwrap/expect are `#[cfg(test)]`). Fix: propagate the error and mark the
  actor `Failed`, mirroring the `StepResult::Error` path.

- **`actor.rs:219` — spawned child actors always get `MockHandler`, never the live handler (MEDIUM, correctness).**
  `let gateway = CapabilityGateway::new(policy)` installs the default `MockHandler`
  (`capability_gateway.rs:438`). The parent's real handler (HTTP/LLM/etc.) is **not** propagated to
  children. Policy *is* cloned so gating still applies, but any capability a child actor performs
  resolves to mock data regardless of the installed live handler. Security-benign (children can't do
  real side effects) but a silent functional trap for multi-actor workflows that expect real effects.

- **`vm.rs:620-711` — arithmetic uses unchecked native ops (`x + y`, `x * y`, `-n`) (MEDIUM).**
  `Op::Add/Sub/Mul/Neg` on `Value::Int(i64)` use raw operators with no `checked_*`/`wrapping_*`
  (grep confirms none in `vm.rs`). In **debug** builds i64 overflow panics → DoS from crafted/looping
  bytecode (e.g. `i64::MAX + 1`, `i64::MIN` negation). In **release** it silently wraps — deterministic
  across platforms but a correctness surprise. `Div`/`Mod` correctly guard zero (`vm.rs:664,685`).

- **`vm.rs:744-748` / `vm.rs:1471-1489` — `EmitUi`, `Debug`, `DebugMsg` bypass the capability gateway (LOW).**
  `EmitUi` pushes to `ui_output` and logs, but performs no `UiRender` capability check.
  `Debug`/`DebugMsg` write to **stderr** — an ungated I/O side effect (documented "operational-only,
  no audit event" at `vm.rs:1472`). Bounded by `max_steps` but not policy-gated; a possible info-leak /
  log-flood vector. Not a determinism problem for audited runs (no EventLog entry).

- **`net_record_replay.rs:118` — crash-during-record loses the tape (LOW, documented).**
  Save-on-`Drop` does not run under `panic = "abort"` / SIGKILL. Author-acknowledged v1 limitation.

## 4. SECURITY (in scope)

### Capability enforcement — mostly complete, double-gated
- **CONFIRMED SAFE:** `Op::CapCall` (`vm.rs:598-619`) enforces **two** independent checks before any
  effect: (1) static — the executing function must statically declare the capability
  (`vm.rs:603-607`, `CapabilityDenied` otherwise); (2) dynamic — `gateway.call` checks `Policy` allow
  and budget (`capability_gateway.rs:489-516`). All effectful handlers (net/fs/db/llm/time/random)
  are reachable **only** through `CapCall`.
- **NEEDS-REVIEW (bypass surface):** effects **not** modeled as `CapCall` skip both gates — `EmitUi`
  (`vm.rs:744`) and `Debug`/`DebugMsg` stderr writes (`vm.rs:1476,1487`). See §3. UI is in-memory;
  stderr is a genuine ungated I/O sink.
- **CONFIRMED SAFE:** budget accounting is `BTreeMap`-keyed and increments before the check
  (`capability_gateway.rs:505-513`); a budget-rejected call is not logged (log happens after the
  check at `:519`), so replay stays consistent.

### SSRF (http_handler.rs, `http` feature only) — guards present but PENETRABLE
- **CONFIRMED bypass — domain names that resolve to private IPs (HIGH).** `validate_url_safety`
  (`http_handler.rs:167`) only calls `is_private_ip` when the host **parses as an IP literal**
  (`:188 if let Ok(ip) = host.parse::<IpAddr>()`). A hostname like `metadata.attacker.com` (or any
  DNS-rebinding host) that resolves to `169.254.169.254` / `127.0.0.1` / `10.x` is **never**
  IP-checked; `ureq` then resolves and connects to the private target. With the **default** empty
  `allowed_domains` (`capability_gateway.rs:59`), the domain allowlist provides no defense either.
  No post-resolution / connect-time IP check exists.
- **CONFIRMED bypass — non-`::1` IPv6 private ranges (HIGH).** `url::Url::host_str()` returns IPv6
  hosts **bracketed** (the code itself relies on this: `host == "[::1]"` at `:183`, `host == "[::0]"`
  at `:196`). But the private-range path at `:188` does `host.parse::<IpAddr>()`, and `IpAddr`
  rejects the bracketed form → parse fails → `is_private_ip`'s entire IPv6 branch (`:213-218`:
  loopback, unspecified, `fc00::/7`) is **unreachable for real URL hosts**. Only the three hardcoded
  string compares (`[::1]`, `[::0]`, `[::]`) block anything. Therefore `http://[fc00::1]/` (ULA),
  `http://[fe80::1]/` (link-local), and IPv4-mapped `http://[::ffff:a9fe:a9fe]/` (→169.254.169.254)
  all pass validation. (Evidence is dispositive: the explicit bracket compares only make sense, and
  `test_ssrf_blocks_ipv6_loopback` at `:304` only passes, because `host_str()` yields brackets.)
- **CONFIRMED bypass — redirects re-target after validation (HIGH, default-on).** `allow_redirects`
  defaults to `true` (`capability_gateway.rs:42,64`) and `HttpHandler::new` only disables redirects
  when false (`http_handler.rs:30`). `validate_url_safety` runs **once** on the initial URL
  (`:52`); `ureq` then follows 3xx `Location` headers with no re-validation. A public URL returning
  `302 → http://169.254.169.254/` reaches the metadata endpoint. Mitigated only by explicitly setting
  `allow_redirects=false`, which is not the default.
- **NEEDS-REVIEW — decimal/hex integer IPv4 hosts.** The comment at `:194` claims to "block numeric
  IPv4 encoded in decimal (e.g. 2130706433)", but the code below only string-matches `0.0.0.0` /
  `[::0]` / `[::]` — it does **not** implement that block. In practice `url` (WHATWG) normalizes
  `http://2130706433/` and `http://0x7f000001/` to `127.0.0.1` before `host_str()`, so the literal
  case is blocked by normalization, not by this code. The comment is misleading/dead; correctness
  depends on `url`'s IPv4 canonicalization rather than an explicit guard.
- **CONFIRMED SAFE:** scheme allowlist rejects non-`http(s)` (`:169-176`); IPv4-literal private/
  loopback/link-local/CGNAT/unspecified are blocked (`:204-212`); response body is size-capped via
  `.take(max_response_bytes)` (`:86`); per-request `timeout_ms` is applied (`:28`).
- **CONFIRMED SAFE (no-net path):** `ReplayingHttpHandler` never touches the network
  (`net_record_replay.rs:293`), so replayed runs have no SSRF surface.

### Panic-on-crafted-bytecode DoS
- **CONFIRMED:** `actor.rs:225` `.expect()` on `SpawnActor` bad index (see §3). Only production panic.
- **CONFIRMED (debug builds):** unchecked integer overflow in `Op::Add/Sub/Mul/Neg` (see §3).
- **CONFIRMED SAFE:** other malformed-index vectors return typed errors, not panics — bad function
  index (`vm.rs:302`), constant (`:393`), global (`:409,415`), local (`:1515,1521`), list/record
  index (`:793,806`), match exhaustion (`:485`). Out-of-range jump targets are handled gracefully:
  `Op::Jmp(offset)` sets `ip=offset` (`:435`) and the next iteration's `ip >= func.code.len()` check
  (`:364`) implicitly returns `Unit` rather than panicking.
- **CONFIRMED SAFE:** the many `self.call_stack.last_mut().unwrap()` (`vm.rs:385,435,479,554`) and
  raw `self.module.functions[func_idx]` indexes (`:363,451,580,603`) are sound — a `CallFrame` exists
  only via `call_function`, which validates `func_idx`, and the frame is re-confirmed at `:352` each
  loop before these run.

### Determinism / nondeterminism leaks
- **CONFIRMED SAFE:** ordered containers throughout — `Policy.rules`/`usage`/`NetPolicy` and
  `Value::Map` are `BTreeMap` (`crates/llmbc/src/value.rs:38`), so `verify_full`'s JSON comparison
  (`replay.rs:225`) and telemetry byte-counts are order-stable. `LlmRouterHandler` provider registry
  is `BTreeMap` (`capability_gateway.rs:323`). No `HashMap` iteration in execution paths.
- **CONFIRMED SAFE:** time/randomness are not read in pure code — they are `Capability::TimeNow` /
  `Capability::Random` routed through `CapCall`→gateway→`EventLog` (recorded for replay). `MockHandler`
  returns fixed values (`capability_gateway.rs:162-163`).
- **CONFIRMED SAFE (well-fenced):** the only wall-clock read (`start.elapsed()`, `vm.rs:339`) yields
  `WallTimeExceeded`, and the code carries an explicit contract (`vm.rs:179-183`) that this path must
  never feed an `EventLog`/`AuditLog`/`EvidenceBundle`; the orchestrator `Runner` deliberately does
  not set it. Treated as an operational guardrail, not a deterministic ceiling (`set_max_steps` is).

### Actor scheduler determinism & supervision
- **CONFIRMED SAFE:** round-robin over a run-queue built in stable actor-index order
  (`actor.rs:129-135`); messages delivered sorted by `(to, from)` via stable `sort_by_key`
  (`actor.rs:259-260`); child IDs assigned from a monotonic `next_id` seeded into each VM
  (`actor.rs:160,209`). Bounded by `max_rounds` (`:127`, default 10_000) and `budget_per_round`
  (`:64`, default 1000) → no unbounded actor execution; `MaxRoundsExceeded`/`Deadlock` are returned.
- **CONFIRMED SAFE:** supervision is OneForOne crash **cascade with no auto-restart** — a failed
  actor sets `Failed`, `cascade_failure` recursively fails descendants (`:276-286`), the parent is
  notified via an injected `Err` message (`:183-191`), spawns/outgoing from the failed actor are
  discarded (`:201-205`), and root failure propagates the error (`:193-195`). `cascade_failure`
  recursion depth = actor-tree depth (bounded by total spawns); no stack-guard, but reachable depth
  is limited by `max_rounds*budget`.

### Step/budget limits — bounded
- **CONFIRMED SAFE:** `execute()` enforces `max_steps` (`vm.rs:329`, default 10_000_000),
  `MAX_STACK=4096` (`vm.rs:1499`), `MAX_CALL_DEPTH=256` (`vm.rs:304`), and per-slice `budget`
  (`vm.rs:346`). A program cannot run unbounded through the normal loop.

### Replay coverage
- **CONFIRMED SAFE:** `ReplayEngine::verify_full` (`replay.rs:214`) compares **every** event type by
  full JSON equality over the whole ordered log (`CapCall`, `CapResult`, `ActorSpawn`, `MessageSend`,
  `MessageReceive`, `UiEmit`, `SchedulerTick`), not just `CapCall`. Serialization uses
  `unwrap_or_default()` (`:225`) so it cannot panic. The lighter `verify` (`:164`) intentionally
  compares `CapCall` only. `EventLog::from_json` rejects unsupported versions (`replay.rs:138`).
